## Pub/Sub
특정한 topic에 대해 해당 topic을 **구독한 모두에게** 메시지를 발행하는 통신 방법

구독자와는 지속적인 TCP 연결을 유지하고 있음
## Pub/Sub의 문제
push 방식이기 때문에 subscriber의 부하를 신경쓰지 않는다.
수신 확인을 하지 않아 100% 전송 보장을 하지 못한다.

## 일반 Key-Value Store와의 차이
1. key-value는 필요한 서버에만 요청을 보낸다.
2. 비정상 종료될 때, key가 그대로 남아있게 된다.
3. 서버가 변경되면, Redis의 매핑 정보를 갱신해주어야 한다.
