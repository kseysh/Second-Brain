## Cache Stampede
캐시에서 특정 데이터가 만료되는 순간, 그 데이터를 조회하려는 많은 요청이 동시에 DB로 몰리면서 시스템 부하가 급격히 상승하는 현상

## 방지법
- Locking: 데이터를 조회하기 전 락을 사용해 단 하나의 요청만 DB에 접근하여 캐시를 갱신하도록 제한하고 다른 요청들은 잠시 대기하게 함
- Jitter: 모든 캐시 데이터의 만료 시간을 동일하게 설정하지 않고, 약간의 무작윈 값을 더해 데이터들이 서로 다른 시점에 만료되도록 분산
- 백그라운드 갱신: 만료 시간이 다 되기 전에 백그라운드 프로세스가 미리 데이터를 갱신하는 방식
- 논리적 만료 사용: 데이터를 논리적으로는 만료시키되, 새로운 데이터가 DB에서 넘어오기 전까지는 옛날 데이터를 그대로 응답
- 확률적 조기 갱신 (PER: Probabilistic Early Recomputation): 캐시의 TTL이 만료되기 전 확률적으로 fallback을 호출하여 캐시 갱신을 수행, TTL이 만료에 가까워질수록 fallback 수행 확률을 높임
