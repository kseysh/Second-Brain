
## 틀린 코드
```sql
SELECT A.AUTHOR_ID, A.AUTHOR_NAME, B.CATEGORY, 
(
    SELECT SUM(BS.SALES) * B.PRICE
    FROM BOOK_SALES AS BS
    WHERE B.BOOK_ID = BS.BOOK_ID AND BS.SALES_DATE BETWEEN '2022-01-01' AND '2022-01-31'
) AS TOTAL_SALES # 기간 내의 book 판매 가격 
FROM AUTHOR AS A, BOOK AS B
WHERE A.AUTHOR_ID = B.AUTHOR_ID
GROUP BY B.CATEGORY
ORDER BY A.AUTHOR_ID ASC, B.CATEGORY DESC;
```

## 정답 코드
```sql
SELECT 
    A.AUTHOR_ID, 
    A.AUTHOR_NAME, 
    B.CATEGORY, 
    SUM(S.SALES * B.PRICE) AS TOTAL_SALES
FROM BOOK B
JOIN AUTHOR A ON B.AUTHOR_ID = A.AUTHOR_ID
JOIN BOOK_SALES S ON B.BOOK_ID = S.BOOK_ID
WHERE S.SALES_DATE BETWEEN '2022-01-01' AND '2022-01-31'
GROUP BY A.AUTHOR_ID, A.AUTHOR_NAME, B.CATEGORY
ORDER BY A.AUTHOR_ID ASC, B.CATEGORY DESC;
```

## 틀린 이유
`JOIN`을 사용한 정답 코드는 '뭉치기(Group By)' 전에 '계산'을 할 수 있는 상태를 만듭니다.
