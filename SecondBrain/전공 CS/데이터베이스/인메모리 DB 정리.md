###### Redis vs. memcached
memcached는 멀티스레드 지원, 적은 메모리 요구 
그에 비해, Redis는 다양한 자료구조를 지원하고, 데이터 영속화 가능
###### Redis의 자료구조
String, Set, List, Sorted Set, Hash, BitArr
###### Redis의 단점
싱글 스레드라 오래걸리는 명령어 실행시 해당 명령이 처리될 때까지 다음 명령어들이 대기해야 한다.
메모리의 특성상 메모리 단편화가 발생한다.
###### Look Aside
데이터를 찾을 때 우선 캐시에 저장된 데이터가 있는지 우선적으로 확인하는 전략
- 장점
	- 원하는 데이터만 캐시에 저장할 수 있다
	- 캐시 장애 대비 구성이 되어 있다
- 단점
	- 캐시가 다운된 순가 DB로 요청이 몰릴 수 있음
###### Read Through
캐시에서만 데이터를 읽어오는 전략
- 장점
	- 캐시와 DB간의 데이터 동기화가 항상 이루어진다.
- 단점
	- 데이터 조회를 캐시에만 의지하므로, redis가 다운될 경우 서비스 이용에 차질이 생길 수 있다
	- 데이터 조회 속도가 느리다
###### Write Around
캐시에 직접 쓰지 않고 DB에 직접 쓴다
- 장점
	- 불필요한 데이터를 캐시에 올리지 않고 바로 써서 리소스를 아낄 수 있다
