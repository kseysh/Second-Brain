## Depends
객체 주입 및 데이터 검증, 인증 체크, DB 세션 생성 등을 함수형으로 연결하는데 최적화
## Spring의 싱글톤 생성자 주입과 다른 점
### Spring
Java는 런타임에 동적으로 함수를 갈아끼우기 어렵기 때문에 미리 인터페이스를 기반으로 프록시 객체를 만들어 놓는 싱글톤 방식이 유리
### FastAPI
Spring처럼 거대한 컨테이너가 미리 모든 객체를 다 만들어 놓는 것이 아닌, Request가 들어오는 시점에 필요한 것들을 준비하는 데 집중함
=> 마이크로서비스 환경에서 각 엔드포인트가 독립적으로 인증이나 DB 세션을 유연하게 다루는 것에 초점을 맞춤
