## DB Driven
데이터베이스 테이블 설계부터 시작하여 테이블 구조에 맞춰 클래스를 만든다.
객체를 데이터 서빙 용도로 사용하며, 비즈니스 로직은 SQL 쿼리나 프로시저에 집중됨
단순 통계 및 Admin 페이지에 사용
### 장점
- 빠른 개발 속도 및 단순함
- SQL 최적화
### 단점
- 객체지향의 장점 포기
- 유지보수 어려움
- 비즈니스 로직 분산
## Transaction Script 
절차지향적 개발 방식
Entity와 Domain은 단순히 Getter, Setter만 지원하고, Service에서 모든 비즈니스 로직을 처리한다.
일반적인 웹 어플리케이션에서 사용
### 장점
- 빠른 개발속도 및 단순함
- 가시성
### 단점
- 서비스 비대화 및 코드 중복

## DDD
비즈니스 로직(도메인)을 소프트웨어의 중심에 두는 설계 방식
도메인 객체가 자신의 로직을 갖는다.
쇼핑몰, 금융 등 복잡한 비즈니스 로직에서 사용
## TDD
DDD와 상호 보완 관계
TDD: 테스트를  통과하기 위한 기능에 집중하여 기능 구현을 우선으로 하는 프로세스
DDD: 도메인의 본질적인 구조와 행동에 집중하여 모델링을 우선으로 하는 프로세스
