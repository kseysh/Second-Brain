#### 메모리 및 리소스 효율성
- 멀티스레드: 스레드 하나당 고유의 스택 메모리 (1MB)
- 비동기 프로그래밍: Task를 가벼운 객체로 관리
#### 제어 흐름과 스케줄링
- 멀티스레드: 선점형 방식
- 비동기 프로그래밍: 협력형 방식
#### I/O 처리 방식
- 멀티스레드: Blocking I/O를 받으면 스레드가 Block함
- 비동기 프로그래밍: Non-Blocking I/O를 사용하여 callee의 작업 완료를 기다리지 않고 콜백을 통해 알려달라고 하며, 즉시 다른 작업 처리
## 비동기 프로그래밍이 멀티스레드보다 I/O Bound 작업에서 유리한 이유
- 스레드 풀 고갈 문제로 비동기 프로그래밍보다 많은 요청을 처리할 수 없음
	- 스레드 풀이 너무 많으면 컨텍스트 스위칭 비용 증가 및 스레드가 메모리를 다 잡아먹음 (스레싱 발생)
- 멀티스레드: OS 레벨 스위칭
	- 레지스터 값, PC, Stack Counter, Stack Pointer 등등 교체
- 비동기 프로그래밍: User 레벨 스위칭
	- 단순히 함수 포인터를 바꾸거나 루프를 한 바퀴 더 도는 수준의 비용
