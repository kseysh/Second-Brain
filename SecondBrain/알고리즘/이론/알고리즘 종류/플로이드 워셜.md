시간복잡도: O(N<sup>3</sup>) (삼중 포문 사용) => N이 500 이하여야 함

```java
for (int k = 0; k < N; k++) {          // 1. 거쳐가는 노드
    for (int i = 0; i < N; i++) {      // 2. 출발 노드
        for (int j = 0; j < N; j++) {  // 3. 도착 노드
            // 기존 경로보다 k를 거쳐가는 경로가 더 짧은 경우 갱신
            if (dist[i][k] + dist[k][j] < dist[i][j]) {
                dist[i][j] = dist[i][k] + dist[k][j];
            }
        }
    }
}
```

## 사용되는 유형

- 사이클(Cycle)의 유무나 음수 사이클을 판별할 때
	- 배열 초기화 시 자기 자신으로 가는 비용 `dist[i][i]`를 0이 아닌 무한대로 둡니다. 플로이드 워셜 수행 후 `dist[i][i]`가 무한대가 아니라면 사이클이 존재하는 것입니다. 만약 `dist[i][i] < 0`인 노드가 있다면 그래프에 음수 사이클이 존재한다는 뜻입니다.
- "왕복 거리" 또는 "특정 노드들을 거쳐가는" 최단 비용을 조합할 때
	- 다익스트라를 여러 번 돌려도 풀리지만, 플로이드 워셜로 `dist` 2차원 배열을 한 번 완성해 두면 로직 처리가 압도적으로 깔끔해지는 유형
- "모든 노드" 간의 관계나 최단 거리를 물어볼 때
	- 특정 출발지가 정해진 것이 아니라, 임의의 출발지와 도착지에 대한 질문이 반복되거나 그 결과들을 조합해야 할 때 사용
